ContentBar Widget Documentation


The ContentBar widget was designed in JavaScript and attempts to provide a simple, reusable mechanism for adding any number of content bars to an HTML Web-page.

Internally, the JavaScript class stores the structure of the content bar as an HTML DOM object. After this is appended to a Web-page you can see that this is a set of HTML tags combined in a special way to look like a content bar. A ContentBar is a block of any HTML content that has been encapsulated.

Although JavaScript must be written to build the widget, it will ultimately be reduced to a set of HTML tags when you view the source code of the Web-page.

Include Mechanism

To use this JavaScript class in a Web-page, please add the HTML script tag with the appropriate attributes to the HEAD section of the HTML document. The path listed assumes that the file is located in the same directory as the HTML Web-page on which the widget will be used (although the include can be made in the BODY of the HTML document it is recommended to place it in the HEAD section to reduce load times). 

<html>
  <head>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
  </head>
<html>

Browser Compatibility

This widget has been tested, and is supported on the following browsers:

  • Firefox 1.5.x
  • Firefox 2.0.x
  • Internet Explorer 6
  • Internet Explorer 7

Class Documentation

  • Attributes:
    • m_id, m_content_class, m_html_content, m_dom
  • Methods:
    • ContentBar(), purgeEvents(), getId(), getDOM(), setContentClass(), setContent(), construct(), destruct()

Attributes

m_id

  The "id" given to the ContentBar when rendered as HTML [Container Node]

m_content_class

  The name of the CSS class that will be given to the ContentBar [Container Node]

m_html_content

  The HTML or text content to be encapsulated within the ContentBar

m_dom

  The ContentBar as an HTML DOM structure

Methods

ContentBar (content_id)

  Construct an HTML ContentBar widget (internally stored as DOM)

  Parameters:
      content_id - the "id" appended to the HTML component [Container Node]
  Notes:
      ContentBar Constructor

purgeEvents ()

  Frees all memory associated with embedded ContentBar events

  Returns:
      void

getId ()

  Returns the "id" of the ContentBar (m_id)

  Returns:
      m_id

getDOM ()

  Returns the DOM representation of the ContentBar

  Returns:
      m_dom

setContentClass (css_class)

  Sets the CSS class for the ContentBar [Container Node]

  Parameters:
      css_class - name of the CSS class to use for the content
  Returns:
      void

setContent (content)

  Sets the HTML or text content for the ContentBar

  Parameters:
      content - a string of HTML or text to make up the ContentBar
  Returns:
      void

construct ()

  Construct the ContentBar as DOM (according to HTML structure)

  Returns:
      a DOM object representing the widget on success
  Notes:
      This must be called each time a change is made to the widget in order to reflect the       corresponding change in HTML

destruct ()

  Free all memory associated with the ContentBar object

  Returns:
      void
  Notes:
      This must be manually called before unloading the Web-page to prevent memory leaks

Example Usage

The examples to follow are designed to serve as a simple walkthrough on how you would add the ContentBar widget to a Web-page. Each example is designed to build on those before it, and therefore, the final example will show the overall picture regarding instantiating, customizing, building, displaying, and destroying the ContentBar widget correctly. While it is not necessary to follow each step exactly, this will provide a general idea on how it works.

Example 1 - Hooking it up

First create an HTML hook or placeholder in which the widget will be placed on the Web-page. While each widget is self-contained, this is an easy way to map out the structure of the page without embedding lots of JavaScript within the HTML code. "div" tags serve as great hooks, since they can be placed within almost any type of tag, and similarly, can contain almost any tag. 

<html>
  <head>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
  </head>
  <body>
    <div id=“content_bar_hook”></div>
  </body>
<html>

Example 2 - Creating the ContentBar

To create the ContentBar, as in other object-oriented programming languages, the constructor must be called along with the use of the keyword new. 

<html>
  <head>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript”>

        var widget = null;

        function init()
        {
            widget = new ContentBar('my_content');
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“content_bar_hook”></div>
  </body>
<html>

Example 3 - Controlling Look and Feel

Some JavaScript must be used in order to customize the colors, borders, and size of the ContentBar. See the class diagram and documentation to determine which methods are available for use. Rather than creating functions to set individual properties as with the ProgressBar widget, because the ContentBar has more properties that can be set, CSS classes are used instead. These stylesheets can be external or internal. For demonstration purposes, internal stylesheet classes will be used. The sample below will create a ContentBar with the formatting described in the CSS classes above, and will feature tabular HTML content. 

<html>
  <head>
    <style type=“text/css”>
      .colored
      {
         background-color:#3C5F84;
         color: #fff;
         width: 600px;	
      }  
    </style>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript”>

        var widget = null;

        function init()
        {	        
            widget = new ContentBar('my_content');
            widget.setContentClass('colored');

            var html = '<table cellspacing="0px" cellpadding="0px">';
            html += '<tr>';
            html += '<td>Timestamp</td>';
            html += '<td>State</td>';
            html += '<td>Formula</td>';
            html += '<td>Value</td>';
            html += '</tr>';        
            html += '</table>';    

            widget.setContent(html);
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“content_bar_hook”></div>
  </body>
<html>

Example 4 - Displaying the ContentBar

To use the widget in a Web-page the construct() method must be called in order to create the HTML DOM representation of the widget. This DOM object must then be appended to the hook that was previously created. This will allow the ContentBar to become visible on the page. If the widget is going to be updated regularly, whether via AJAX or by having the user perform an action such as clicking a button, the construct() method must be called each time and the widget must be appended to the hook each time. Clear the hook of any previously appended ContentBar widgets if you only want one to appear on the Web-page (that is, set the innerHTML property of the hook to an empty string). 

<html>
  <head>
    <style type=“text/css”>
      .colored
      {
         background-color:#3C5F84;
         color: #fff;
         width: 600px;	
      }  
    </style>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript”>

        var widget = null;

        function init()
        {	        
            widget = new ContentBar('my_content');
            widget.setContentClass('colored');

            var html = '<table cellspacing="0px" cellpadding="0px">';
            html += '<tr>';
            html += '<td>Timestamp</td>';
            html += '<td>State</td>';
            html += '<td>Formula</td>';
            html += '<td>Value</td>';
            html += '</tr>';        
            html += '</table>';    

            widget.setContent(html);

            var hook = document.getElementById('content_bar_hook');
            hook.innerHTML = '';
            hook.appendChild(widget.construct());            
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“content_bar_hook”></div>
  </body>
<html>

Example 5 - Cleaning up Memory

To ensure that all memory associated with the widget is freed upon closing the Web-page or browser, it is necessary to call the destruct() method. While most programming languages have a destructor that is implicitly called, here, a explicit function call must be made. The hook to which the widget was appended should also be cleared. 

<html>
  <head>
    <style type=“text/css”>
      .colored
      {
         background-color:#3C5F84;
         color: #fff;
         width: 600px;	
      }  
    </style>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript”>

        var widget = null;

        function init()
        {	        
            widget = new ContentBar('my_content');
            widget.setContentClass('colored');

            var html = '<table cellspacing="0px" cellpadding="0px">';
            html += '<tr>';
            html += '<td>Timestamp</td>';
            html += '<td>State</td>';
            html += '<td>Formula</td>';
            html += '<td>Value</td>';
            html += '</tr>';        
            html += '</table>';    

            widget.setContent(html);

            var hook = document.getElementById('content_bar_hook');
            hook.innerHTML = '';
            hook.appendChild(widget.construct());            
        }

        function destroy()
        {
            widget.destruct();
            document.getElementById('content_bar_hook').innerHTML = '';	        
        }
    </script>
  </head>
  <body onload=“init()” unload=“destroy()”>
    <div id=“content_bar_hook”></div>
  </body>
<html>

Special Notes

  • This particular widget does not use default style in the absence of CSS classes.
  • Always call the destruct() method when unloading the page to prevent memory leaks.
  • The widget is designed to be used with the ContentPane widget rather than as a standalone component.

Lessons Learned

When the widget was created the overall goal was to create a logical structure with which to build an Accordion widget. The rationale was that an Accordion widget would be composed of ContentPanes, which were made up of a TitleBar and a ContentBar. While the TitleBar structure proved its worth, the ContentBar was not necessary since it is only an HTML div tag with encapsulated content.

Although the ContentBar structure serves its purpose, future versions of the widgets will eliminate the ContentBar widget to improve performance.
