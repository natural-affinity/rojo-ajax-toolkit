Dashboard Widget Documentation


The Dashboard widget was designed in JavaScript and attempts to provide a simple, reusable mechanism for adding any number of Dashboards to an HTML Web-page.

Internally, the JavaScript class stores the structure of the dashboard as an HTML DOM object. After this is appended to a Web-page, you can see that this is a set of HTML tags combined in a special way to look like a dashboard. A Dashboard is a set of tabular data (for example, system status and diagnostic information) that is organized into two halves, where each half contains a set of label-value pairs. The structure provides a place to store and display information in an organized and aesthetically pleasing manner.

The Dashboard widget makes use of the DataSet widget for organizing content prior to creation and formatting of the data. The only restriction is that the DataSet object must have exactly four columns in the following sequence: Label, Value, Label, Value. This ensures a level of consistency when displaying the widget. In future, this restriction might be removed to allow for a greater variety of dashboard displays. While there is a column restriction, a limit has not been set for the number of rows of data that be inserted.

Although JavaScript must be written to build the widget, it will ultimately be reduced to a set of HTML tags when you view the source code of the Web-page.

Include Mechanism

To use this JavaScript class in a Web-page, please add the HTML script tag with the appropriate attributes to the HEAD section of the HTML document. The path listed assumes that the file is located in the same directory as the HTML Web-page on which the widget will be used (although the include can be made in the BODY of the HTML document it is recommended to place it in the HEAD section to reduce load times). The dependency widget class (that is, the DataSet) must also be included. 

<html>
  <head>
    <script type=“text/javascript” src=“./class.dataset.js”></script>
    <script type=“text/javascript” src=“./class.dashboard.js”></script>
  </head>
<html>

Browser Compatibility

This widget has been tested, and is supported on the following browsers:

  • Firefox 1.5.x
  • Firefox 2.0.x
  • Internet Explorer 6
  • Internet Explorer 7

HTML Structure

<div>                            <!-- Container Node -->
  <table>                        <!-- Table Node -->
    <thead></thead>
    <tbody>
      [<tr>                      <!-- Row Nodes -->
         <td>...</td>            <!-- Label Node -->
         <td>...</td>            <!-- Value Node -->
         <td></td>               <!-- Space Node -->
         <td>...</td>            <!-- Label Node -->
         <td>...</td>            <!-- Value Node -->
      </tr>]*
    </tbody>
  </table>
</div>

Attributes

m_id

  The "id" given to the Dashboard when rendered as HTML [Container Node]

m_container_class

  The name of the CSS class applied to the Dashboard [Container Node]

m_table_class

  The name of the CSS class applied to the Dashboard [Table Node]

m_row_class

  The name of the CSS class applied to the Dashboard rows [Row Node]

m_row_alt_class

  The name of the CSS class applied to the alternate Dashboard rows [Row Node]

m_label_class

  The name of the CSS class applied to the Dashboard label columns [Label Node]

m_value_class

  The name of the CSS class applied to the Dashboard value columns [Value Node]

m_table_width

  Sets the width of the Dashboard [Container Node]

m_space_width

  Sets the width of the space (column) between the Dashboard halves [Space Node]

m_dom

  Stores the Dashboard as an HTML DOM structure

Example Usage

The examples to follow are designed to serve as a simple walkthrough on how you would add the Dashboard widget to a Web-page. Each example is designed to build on those before it, and hence, the final example will show the overall picture regarding instantiating, customizing, building, displaying, and destroying the Dashboard widget correctly. While it is not necessary to follow each step exactly, this will provide a general idea on how it works.

Example 1 - Hooking it up

First create an HTML hook or placeholder in which the widget will be placed on the Web-page. While each widget is self-contained, this is an easy way to map out the structure of the page without embedding lots of JavaScript within the HTML. "div" tags serve as great hooks, since they can be placed within almost any type of tag, and similarly, can contain almost any tag. 

<html>
  <head>
    <script type=“text/javascript” src=“./class.dataset.js”></script>
    <script type=“text/javascript” src=“./class.dashboard.js”></script>
  </head>
  <body>
    <div id=“dashboard_hook”></div>
  </body>
<html>

Example 2 - Creating the Dashboard

To create the Dashboard, as in other object-oriented programming languages, the constructor must be called along with the use of the keyword new. Remember to create the DataSet widget required for the building the dash content. 

<html>
  <head>
    <script type=“text/javascript” src=“./class.dataset.js”></script>
    <script type=“text/javascript” src=“./class.dashboard.js”></script>
    <script type=“text/javascript”>

        var widget = null;
        var data = null;

        function init()
        {
            widget = new Dashboard('my_dash');
            data = new DataSet(); 
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“dashboard_hook”></div>
  </body>
<html>

Example 3 - Controlling Look and Feel

Some JavaScript must be used in order to customize the colors, borders, and size of the Dashboard widget. See the class diagram and documentation to determine which methods are available for use. Rather than creating functions to set individual properties as with the ProgressBar widget, because the Dashboard has more properties that can be set, CSS classes are used instead. These stylesheets can be external or internal. For demonstration purposes, internal stylesheet classes will be used. The example will create a Dashboard with the formatting described below. The Dashboard width and spacing width must be set. 

<html>
  <head>
    <style type=“text/css”
      table.secondary
      {
        width:800px;
        border:0px;	
      }

      tr.ibm2
      {
        background-color: #C7E4C0; 
        color:black;	
      }

      tr.ibm2,
      tr.ibm3
      {
        font-family: helvetica, arial, sans-serif;
        font-size:12px;	
      }

      .space2
      {
        padding-left:2px;	
      }

      .space10
      {
        padding-left:10px;	
        font-weight:bold;
      }    
    </style>
    <script type=“text/javascript” src=“./class.dataset.js”></script>
    <script type=“text/javascript” src=“./class.dashboard.js”></script>
    <script type=“text/javascript”>

        var widget = null;
        var data = null;

        function init()
        {
            widget = new Dashboard('my_dash');
            data = new DataSet(); 

            widget.setTableClass('secondary');
            widget.setRowClass('ibm3');
            widget.setRowAltClass('ibm2');
            widget.setLabelClass('space2');
            widget.setValueClass('space10');
            widget.setTableWidth(800);
            widget.setSpaceWidth(4);

            data.addDataRow(['Label1','Value1','Label2','Value2']);
            data.addDataRow(['Label3','Value3','Label4','Value4']);
            data.addDataRow(['Label5','Value5','Label6','Value6']);
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“dashboard_hook”></div>
  </body>
<html>

Example 4 - Displaying the Dashboard

To use the widget in a Web-page the construct() method must be called in order to create the HTML DOM representation of the widget. This DOM object must then be appended to the hook that was previously created. This will allow the Dashboard to become visible on the page. If the widget is going to be updated regularly, whether via AJAX or by having the user perform an action such as clicking a button, the construct() method must be called each time and the widget must be appended to the hook each time. Clear the hook of any previously appended Dashboard widgets if you only want one to appear on the Web-page (that is, set the innerHTML property of the hook to an empty string). 

<html>
  <head>
    <style type=“text/css”
      table.secondary
      {
        width:800px;
        border:0px;	
      }

      tr.ibm2
      {
        background-color: #C7E4C0; 
        color:black;	
      }

      tr.ibm2,
      tr.ibm3
      {
        font-family: helvetica, arial, sans-serif;
        font-size:12px;	
      }

      .space2
      {
        padding-left:2px;	
      }

      .space10
      {
        padding-left:10px;	
        font-weight:bold;
      }    
    </style>
    <script type=“text/javascript” src=“./class.dataset.js”></script>
    <script type=“text/javascript” src=“./class.dashboard.js”></script>
    <script type=“text/javascript”>

        var widget = null;
        var data = null;

        function init()
        {
            widget = new Dashboard('my_dash');
            data = new DataSet(); 

            widget.setTableClass('secondary');
            widget.setRowClass('ibm3');
            widget.setRowAltClass('ibm2');
            widget.setLabelClass('space2');
            widget.setValueClass('space10');
            widget.setTableWidth(800);
            widget.setSpaceWidth(4);

            data.addDataRow(['Label1','Value1','Label2','Value2']);
            data.addDataRow(['Label3','Value3','Label4','Value4']);
            data.addDataRow(['Label5','Value5','Label6','Value6']);

            var hook = document.getElementById('dashboard_hook');
            hook.innerHTML = '';
            hook.appendChild(widget.construct(data));              
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“dashboard_hook”></div>
  </body>
<html>

Example 5 - Cleaning up Memory

To ensure that all memory associated with the widget is freed upon closing the Web-page or browser, it is necessary to call the destruct() method. While most programming languages have a destructor that is implicitly called, here, a explicit function call must be made. The hook to which the widget was appended should also be cleared. 

<html>
  <head>
    <style type=“text/css”
      table.secondary
      {
        width:800px;
        border:0px;	
      }

      tr.ibm2
      {
        background-color: #C7E4C0; 
        color:black;	
      }

      tr.ibm2,
      tr.ibm3
      {
        font-family: helvetica, arial, sans-serif;
        font-size:12px;	
      }

      .space2
      {
        padding-left:2px;	
      }

      .space10
      {
        padding-left:10px;	
        font-weight:bold;
      }    
    </style>
    <script type=“text/javascript” src=“./class.dataset.js”></script>
    <script type=“text/javascript” src=“./class.dashboard.js”></script>
    <script type=“text/javascript”>

        var widget = null;
        var data = null;

        function init()
        {
            widget = new Dashboard('my_dash');
            data = new DataSet(); 

            widget.setTableClass('secondary');
            widget.setRowClass('ibm3');
            widget.setRowAltClass('ibm2');
            widget.setLabelClass('space2');
            widget.setValueClass('space10');
            widget.setTableWidth(800);
            widget.setSpaceWidth(4);

            data.addDataRow(['Label1','Value1','Label2','Value2']);
            data.addDataRow(['Label3','Value3','Label4','Value4']);
            data.addDataRow(['Label5','Value5','Label6','Value6']);

            var hook = document.getElementById('dashboard_hook');
            hook.innerHTML = '';
            hook.appendChild(widget.construct(data));              
        }

        function destroy()
        {
            widget.destruct();
            document.getElementById('dashboard_hook').innerHTML = '';                	        
        }
    </script>
  </head>
  <body onload=“init()” unload=“destroy()”>
    <div id=“dashboard_hook”></div>
  </body>
<html>

Special Notes

  • This particular widget does not use default style in the absence of CSS classes.
  • Always call the destruct() method when unloading the page to prevent memory leaks.
  • The DataSet object passed in upon construct() must be exactly four columns, but can have any number of rows.
  • Since a DataSet object can theoretically accept any data, including HTML, it is possible to embed other widgets within the Dashboard. However, if these widgets contain events, the dashboard will not remove these upon calling the destruct() method. It is up to the programmer to do so.
  • If the width of the content placed into a particular dashboard column exceeds the maximum value defined in the formulas above (See Look and Feel Diagram), then the length of the column may be extended and it is possible that the Dashboard width might be greater than specified.

Lessons Learned

While it was not necessary to make the Dashboard into a widget, it helped to provide a consistency in terms how data was stored and organized throughout the entire application (that is, all data is stored in a widget).

Although using the DataSet widget to prepare the Dashboard data may seem more diffcult than using a straight HTML dashboard, it allows for a level of consistency amongst all of the widgets.

While the Dashboard widget could have been written to accomodate any number of columns rather than just rows, this was much simpler in terms of calculations and considering possibilities where errors could occur.
