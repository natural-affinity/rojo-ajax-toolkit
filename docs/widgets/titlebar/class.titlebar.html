TitleBar Widget Documentation


The TitleBar widget was designed in JavaScript and attempts to provide a simple, reusable mechanism for adding any number of TitleBars to an HTML Web-page.

Internally, the JavaScript class stores the structure of the title bar as an HTML DOM object. After this is appended to a Web-page you can see that this is simply a set of HTML tags combined in a special way to look like a title bar. A TitleBar is a single row of information (possibly with a colored background embedded images or icons).

Although JavaScript must be written to build the widget, it will ultimately be reduced to a set of HTML tags when you view the source code of the Web-page.

Include Mechanism

To use this JavaScript class in a Web-page, please add the HTML script tag with the appropriate attributes to the HEAD section of the HTML document. The path listed assumes that the class file is located in the same directory as the HTML Web-page on which the widget will be used (although the include can be made in the BODY of the HTML document, it is recommended to place it in the HEAD section to reduce load times). 

<html>
  <head>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
  </head>
<html>

Browser Compatibility

This widget has been tested, and is supported on the following browsers:

  • Firefox 1.5.x
  • Firefox 2.0.x
  • Internet Explorer 6
  • Internet Explorer 7

HTML Structure

<div>                            <!-- Container Node -->
  <table>                        <!-- Table Node -->
    <thead></thead>
    <tbody>
      <tr>                       <!-- Row Node -->
        [<td>...</td>]*          <!-- Column Nodes -->
      </tr>
    </tbody>
  </table>
</div>

Class Documentation

  • Attributes:
    • m_id, m_container_class, m_table_class, m_row_class, m_col_class, m_html_content, m_dom
  • Methods:
    • TitleBar(), purgeEvents(), getId(), getDOM(), setContainerClass(), setTableClass(), setRowClass(), setColClass(), setContent(), construct(), destruct()

Attributes

m_id

  The "id" given to the TitleBar when rendered as HTML [Container Node]

m_container_class

  The name of the CSS class that will be given to the TitleBar [Container Node]

m_table_class

  The name of the CSS class that will be given to the TitleBar [Table Node]

m_row_class

  The name of the CSS class that will be given to the TitleBar [Row Node]

m_col_class

  The name of the CSS classes that will apply to each of the TitleBar columns [Column Nodes]

m_html_content

  An array of HTML content to split across the TitleBar as columns [ Column Nodes]

m_dom

  The TitleBar as an HTML DOM structure

Methods

TitleBar (title_id)

  Construct an HTML TitleBar widget (internally stored as DOM)

  Parameters:
      title_id - the "id" appended to the HTML component [Container Node]
  Notes:
      TitleBar Constructor

purgeEvents ()

  Frees all memory associated with embedded TitleBar events

  Returns:
      void

getId ()

  Returns the "id" of the TitleBar (m_id)

  Returns:
      m_id

getDOM ()

  Returns the DOM representation of the TitleBar

  Returns:
      m_dom

setContainerClass (css_class)

  Sets the CSS Class for the TitleBar [Container Node]

  Parameters:
      css_class - name of the CSS class to use
  Returns:
      void

setTableClass (css_class)

  Sets the CSS class for the TitleBar [Table Node]

  Parameters:
      css_class - name of the CSS class to use
  Returns:
      void

setRowClass (css_class)

  Sets the CSS class for the TitleBar [Row Node]

  Parameters:
      css_class - name of the CSS class to use
  Returns:
      void

setColClass (css_class)

  Sets the CSS classes for the TitleBar [Column Nodes]

  Parameters:
      css_class - an array of names of the CSS classes to use
  Returns:
      void

setContent (content)

  Sets the HTML or text content for the TitleBar columns

  Parameters:
      content - an array of HTML or text content to make up the TitleBar columns
  Returns:
      void

construct ()

  Construct the TitleBar as DOM (according to HTML structure)

  Returns:
      a DOM object representing the widget on success
  Notes:
      This must be called each time a change is made to the widget in order to reflect the       corresponding change in HTML

destruct ()

  Free all memory associated with the TitleBar object

  Returns:
      void
  Notes:
      This must be manually called before unloading the Web-page to prevent memory leaks

Example Usage

The examples to follow are designed to serve as a simple walkthrough on how you would add the TitleBar widget to a Web-page. Each example is designed to build on those before it, and therefore, the final example will show the overall picture regarding instantiating, customizing, building, displaying, and destroying the TitleBar widget correctly. While it is not necessary to follow each step exactly, this will provide a general idea on how it works.

Example 1 - Hooking it up

First create an HTML hook or placeholder in which the widget will be placed on the Web-page. While each widget is self-contained, this is an easy way to map out the structure of the page without embedding lots of JavaScript within the HTML. "div" tags serve as great hooks, since they can be placed within almost any type of tag, and similarly, can contain almost any tag. 

<html>
  <head>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
  </head>
  <body>
    <div id=“title_bar_hook”></div>
  </body>
<html>

Example 2 - Creating the TitleBar

To create the TitleBar, as in other object-oriented programming languages, the constructor must be called along with the use of the keyword new. 

<html>
  <head>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript”>

        var widget = null;

        function init()
        {
            widget = new TitleBar('my_title');
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“title_bar_hook”></div>
  </body>
<html>

Example 3 - Controlling Look and Feel

Some JavaScript must be used in order to customize the colors, borders, and size of the TitleBar. See the class diagram and documentation to determine which methods are available for use. Rather than creating functions to set individual properties as with the ProgressBar widget, since the TitleBar has more properties that can be set, CSS classes are used instead. These stylesheets can be external or internal. For demonstration purposes, internal stylesheet classes will be used. The example below will create a TitleBar with the formatting described in the CSS classes above and will have two columns of content: one is HTML based and embeds an image while the other is the text for the title. 

<html>
  <head>
    <style type=“text/css”>
      .label
      {
         background-color:#3C5F84;
         color: #fff;
         font-family: verdana,arial,sans-serif;
         font-size:11px;
         font-weight:bold;
         padding-left:2px;
         height:22px;	
      }  

      td.fixed
      {
         width:758px;   
      }
    </style>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript”>

        var widget = null;

        function init()
        {
            widget = new TitleBar('my_title');
            widget.setContainerClass('label');
            widget.setRowClass('label');
            widget.setColClass('','fixed');
            widget.setContent(['<img src=“./s.jpg” />',' Example - ']);
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“title_bar_hook”></div>
  </body>
<html>

Example 4 - Displaying the TitleBar

To use the widget in a Web-page the construct() method must be called in order to create the HTML DOM representation of the widget. This DOM object must then be appended to the hook that was previously created. This will allow the TitleBar to become visible on the page. If the widget is going to be updated regularly, whether via AJAX or by having the user perform an action such as clicking a button, the construct() method must be called each time and the widget must be appended to the hook each time. Clear the hook of any previously appended TitleBar widgets if you only want one to appear on the Web-page (that is, set the innerHTML property of the hook to an empty string). 

<html>
  <head>
    <style type=“text/css”>
      .label
      {
         background-color:#3C5F84;
         color: #fff;
         font-family: verdana,arial,sans-serif;
         font-size:11px;
         font-weight:bold;
         padding-left:2px;
         height:22px;	
      }  

      td.fixed
      {
         width:758px;   
      }
    </style>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript”>

        var widget = null;

        function init()
        {
            widget = new TitleBar('my_title');
            widget.setContainerClass('label');
            widget.setRowClass('label');
            widget.setColClass('','fixed');
            widget.setContent(['<img src=“./s.jpg” />',' Example - ']);

            var hook = document.getElementById('title_bar_hook');
            hook.innerHTML = '';
            hook.appendChild(widget.construct());
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“title_bar_hook”></div>
  </body>
<html>

Example 5 - Cleaning up Memory

To ensure that all memory associated with the widget is freed upon closing the Web-page or browser it is necessary to call the destruct() method. While most programming languages have a destructor that is implicitly called, here, a explicit function call must be made. The hook to which the widget was appended should also be cleared. 

<html>
  <head>
    <style type=“text/css”>
      .label
      {
         background-color:#3C5F84;
         color: #fff;
         font-family: verdana,arial,sans-serif;
         font-size:11px;
         font-weight:bold;
         padding-left:2px;
         height:22px;	
      }  

      td.fixed
      {
         width:758px;   
      }
    </style>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript”>

        var widget = null;

        function init()
        {
            widget = new TitleBar('my_title');
            widget.setContainerClass('label');
            widget.setRowClass('label');
            widget.setColClass('','fixed');
            widget.setContent(['<img src=“./s.jpg” />',' Example - ']);

            var hook = document.getElementById('title_bar_hook');
            hook.innerHTML = '';
            hook.appendChild(widget.construct());
        }

        function destroy()
        {
            widget.destruct();
            document.getElementById('title_bar_hook').innerHTML = '';    
        }
    </script>
  </head>
  <body onload=“init()” unload=“destroy()”>
    <div id=“title_bar_hook”></div>
  </body>
<html>

Special Notes

  • This particular widget does not use default styles in the absence of CSS classes.
  • Always call the destruct() method when unloading the page to prevent memory leaks.

Lessons Learned

The only complication surrounding the TitleBar widget was the need to apply the same CSS class to both the row and container nodes. This ensures cross-browser compatibility is achieved.
