ContentPane Widget Documentation


The ContentPane widget was designed in JavaScript and attempts to provide a simple, reusable mechanism for adding any number of content panes to an HTML Web-page.

Internally, the JavaScript class stores the structure of the content pane as an HTML DOM object. After this is appended to a Web-page you can see that it is a set of HTML tags combined in a special way to look like a content pane. A ContentPane is made of two sub-components: a TitleBar, and a ContentBar. Together, they appear as a section of content with a header.

Although JavaScript must be written to build the widget, it will ultimately be reduced to a set of HTML tags when you view the source code of the Web-page. Even though this widget contains two other widgets, the widgets still breakdown into their respective HTML structures. The ContentPane guarantees that the HTML will be formatted as a TitleBar on top of a ContentBar.

Include Mechanism

To use this JavaScript class in a Web-page, please add the HTML script tag with the appropriate attributes to the HEAD section of the HTML document. The path listed assumes that the file is located in the same directory as the HTML Web-page on which the widget will be used (although the include can be made in the BODY of the HTML document it is recommended to place it in the HEAD section to reduce load times). The dependency widget classes (that is, TitleBar and ContentBar) must also be included. 

<html>
  <head>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
  </head>
<html>

Browser Compatibility

This widget has been tested, and is supported on the following browsers:

  • Firefox 1.5.x
  • Firefox 2.0.x
  • Internet Explorer 6
  • Internet Explorer 7

HTML Structure

<div>                            <!-- Pane Node -->
  <div>                          <!-- TitleBar -->
    <table>                        
      <thead></thead>
      <tbody>
        <tr>                       
          [<td>...</td>]*          
        </tr>
      </tbody>
    </table>
  </div>
  <div>                          <!-- ContentBar -->  
    ...                            
  </div>
</div>

Class Documentation

  • Attributes:
    • m_id, m_pane_class, m_title_bar, m_content_bar, m_dom
  • Methods:
    • ContentPane(), purgeEvents(), getId(), getDOM(), setTitleBar(), setContentBar(), setPaneClass(), construct(), destruct()

Attributes

m_id

  The "id" given to the ContentPane when rendered as HTML [Pane Node]

m_pane_class

  The name of the CSS class given to the ContentPane [Pane Node]

m_title_bar

  The TitleBar object that will appear at the top of the ContentPane

m_content_bar

  The ContentBar object that will appear at the bottom of the ContentPane

m_dom

  Stores the ContentPane as an HTML DOM structure

Methods

ContentPane (pane_id)

  Construct an HTML ContentPane widget (internally stored as DOM)

  Parameters:
      pane_id - the "id" appended to the HTML component [Pane Node]
  Notes:
      ContentPane Constructor

purgeEvents ()

  Frees all memory associated with embedded ContentPane events

  Returns:
      void

getId ()

  Returns the "id" of the ContentPane (m_id)

  Returns:
      m_id

getDOM ()

  Returns the DOM representation of the ContentPane

  Returns:
      m_dom

setTitleBar (obj)

  Sets the TitleBar for the ContentPane

  Parameters:
      obj - the TitleBar object contained in the ContentPane
  Returns:
      void

setContentBar (obj)

  Sets the ContentBar for the ContentPane

  Parameters:
      obj - the ContentBar object contained in the ContentPane
  Returns:
      void

setPaneClass(css_class)

  Sets the CSS class for the ContentPane [Pane Node]

  Parameters:
      css_class - name of the CSS class to use for the pane
  Returns:
      void

construct ()

  Construct the ContentPane as DOM (according to HTML structure)

  Returns:
      a DOM object representing the widget on success
  Notes:
      This must be called each time a change is made to the widget in order to reflect the       corresponding change in HTML

destruct ()

  Free all memory associated with the ContentPane object, including sub-widgets contained within
  the ContentPane

  Returns:
      void
  Notes:
      This must be manually called before unloading the Web-page to prevent memory leaks

Example Usage

The examples to follow are designed to serve as a simple walkthrough on how you would add the ContentPane widget to a Web-page. Each example is designed to build on those before it, and therefore, the final example will show the overall picture regarding instantiating, customizing, building, displaying, and destroying the ContentPane widget correctly. While it is not necessary to follow each step exactly, this will provide a general idea on how it works.

Example 1 - Hooking it up

First create an HTML hook or placeholder in which the widget will be placed on the Web-page. While each widget is self-contained, this is an easy way to map out the structure of the page without embedding lots of JavaScript within the HTML. "div" tags serve as great hooks, since they can be placed within almost any type of tag, and similarly, can contain almost any tag. Although this widget contains sub-widgets, only one hook needs to be created for the parent component. 

<html>
  <head>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
  </head>
  <body>
    <div id=“content_pane_hook”></div>
  </body>
<html>

Example 2 - Creating the ContentPane

To create the ContentPane, as in other object-oriented programming languages, the constructor must be called along with the use of the keyword new. Remember to create the sub-widgets required for the building the parent widget. For this widget, a TitleBar, ContentBar, and ContentPane widget must be created. 

<html>
  <head>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
    <script type=“text/javascript”>

        var widget = null;
        var sub_widget1 = null;
        var sub_widget2 = null;

        function init()
        {
            sub_widget1 = new TitleBar('my_title');
            sub_widget2 = new ContentBar('my_content');
            widget = new ContentPane('my_pane');
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“content_pane_hook”></div>
  </body>
<html>

Example 3 - Controlling Look and Feel

Some JavaScript must be used in order to customize the colors, borders, and size of the TitleBar, ContentBar, and ContentPane. See the class diagram and documentation to determine which methods are available for use. Rather than creating functions to set individual properties as with the ProgressBar widget, because the TitleBar, ContentBar, and ContentPane have more properties that can be set, CSS classes are used instead. These stylesheets can be external or internal. For demonstration purposes, internal stylesheet classes will be used. The example below will create a TitleBar, ContentBar, and ContentPane with the formatting described below. The TitleBar and ContentBar objects must also be attached to the ContentPane. 

<html>
  <head>
    <style type=“text/css”>
      .label
      {
         background-color:#3C5F84;
         color: #fff;
         font-family: verdana,arial,sans-serif;
         font-size:11px;
         font-weight:bold;
         padding-left:2px;
         height:22px;	
      }  

      td.fixed
      {
         width:758px;   
      }

      .colored
      {
         background-color:#3C5F84;
         color: #fff;
         width: 600px;	
      }        
    </style>  
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
    <script type=“text/javascript”>

        var widget = null;
        var sub_widget1 = null;
        var sub_widget2 = null;

        function init()
        {
          sub_widget1 = new TitleBar('my_title');
          sub_widget1.setContainerClass('label');
          sub_widget1.setRowClass('label');
          sub_widget1.setColClass('','fixed');
          sub_widget1.setContent(['<img src=“./s.jpg” />',' Example - ']);            

          sub_widget2 = new ContentBar('my_content');
          sub_widget2.setContentClass('colored');            

          var html = '<table cellspacing="0px" cellpadding="0px">';
          html += '<tr>';
          html += '<td>Timestamp</td>';
          html += '<td>State</td>';
          html += '<td>Formula</td>';
          html += '<td>Value</td>';
          html += '</tr>';        
          html += '</table>';    

          sub_widget2.setContent(html);            

          widget = new ContentPane('my_pane');   
          widget.setTitleBar(sub_widget1);
          widget.setContentBar(sub_widget2);              
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“content_pane_hook”></div>
  </body>
<html>

Example 4 - Displaying the ContentPane

To use the widget in a Web-page the construct() method must be called in order to create the HTML DOM representation of the widget. This DOM object must then be appended to the hook that was previously created. This will allow the ContentPane to become visible on the page. If the widget is going to be updated regularly, whether via AJAX or by having the user perform an action such as clicking a button, the construct() method must be called each time and the widget must be appended to the hook each time. Clear the hook of any previously appended ContentPane widgets if you only want one to appear on the Web-page (that is, set the innerHTML property of the hook to an empty string). All sub-widget construct() method calls will be performed automatically. 

<html>
  <head>
    <style type=“text/css”>
      .label
      {
         background-color:#3C5F84;
         color: #fff;
         font-family: verdana,arial,sans-serif;
         font-size:11px;
         font-weight:bold;
         padding-left:2px;
         height:22px;	
      }  

      td.fixed
      {
         width:758px;   
      }

      .colored
      {
         background-color:#3C5F84;
         color: #fff;
         width: 600px;	
      }        
    </style>  
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
    <script type=“text/javascript”>

        var widget = null;
        var sub_widget1 = null;
        var sub_widget2 = null;

        function init()
        {
          sub_widget1 = new TitleBar('my_title');
          sub_widget1.setContainerClass('label');
          sub_widget1.setRowClass('label');
          sub_widget1.setColClass('','fixed');
          sub_widget1.setContent(['<img src=“./s.jpg” />',' Example - ']);            

          sub_widget2 = new ContentBar('my_content');
          sub_widget2.setContentClass('colored');            

          var html = '<table cellspacing="0px" cellpadding="0px">';
          html += '<tr>';
          html += '<td>Timestamp</td>';
          html += '<td>State</td>';
          html += '<td>Formula</td>';
          html += '<td>Value</td>';
          html += '</tr>';        
          html += '</table>';    

          sub_widget2.setContent(html);            

          widget = new ContentPane('my_pane');   
          widget.setTitleBar(sub_widget1);
          widget.setContentBar(sub_widget2); 

          var hook = document.getElementById('content_pane_hook');
          hook.innerHTML = '';
          hook.appendChild(widget.construct());                       
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“content_pane_hook”></div>
  </body>
<html>

Example 5 - Cleaning up Memory

To ensure that all memory associated with the widget is freed upon closing the Web-page or browser, it is necessary to call the destruct() method. While most programming languages have a destructor that is implicitly called, here, a explicit function call must be made. The hook to which the widget was appended should also be cleared. The destruct() method of the parent widget automatically calls the destruct() method of the sub-widgets. 

<html>
  <head>
    <style type=“text/css”>
      .label
      {
         background-color:#3C5F84;
         color: #fff;
         font-family: verdana,arial,sans-serif;
         font-size:11px;
         font-weight:bold;
         padding-left:2px;
         height:22px;	
      }  

      td.fixed
      {
         width:758px;   
      }

      .colored
      {
         background-color:#3C5F84;
         color: #fff;
         width: 600px;	
      }        
    </style>  
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
    <script type=“text/javascript”>

        var widget = null;
        var sub_widget1 = null;
        var sub_widget2 = null;

        function init()
        {
          sub_widget1 = new TitleBar('my_title');
          sub_widget1.setContainerClass('label');
          sub_widget1.setRowClass('label');
          sub_widget1.setColClass('','fixed');
          sub_widget1.setContent(['<img src=“./s.jpg” />',' Example - ']);            

          sub_widget2 = new ContentBar('my_content');
          sub_widget2.setContentClass('colored');            

          var html = '<table cellspacing="0px" cellpadding="0px">';
          html += '<tr>';
          html += '<td>Timestamp</td>';
          html += '<td>State</td>';
          html += '<td>Formula</td>';
          html += '<td>Value</td>';
          html += '</tr>';        
          html += '</table>';    

          sub_widget2.setContent(html);            

          widget = new ContentPane('my_pane');   
          widget.setTitleBar(sub_widget1);
          widget.setContentBar(sub_widget2); 

          var hook = document.getElementById('content_pane_hook');
          hook.innerHTML = '';
          hook.appendChild(widget.construct());                       
        }

        function destroy()
        {
          widget.destruct();
          document.getElementById('content_pane_hook').innerHTML = '';    
        }        
    </script>
  </head>
  <body onload=“init()” unload=“destroy()”>
    <div id=“content_pane_hook”></div>
  </body>
<html>

Special Notes

  • This particular widget does not use default style in the absence of CSS classes.
  • Always call the destruct() method when unloading the page to prevent memory leaks.
  • Sub-widgets must be created and attached in order to use the widget correctly.
  • Sub-widget construct() and destruct() method calls are automatically managed by the parent (ContentPane) widget.

Lessons Learned

While the ContentPane can be used on its own on a Web-page, its real use is for the Accordion widget. It is designed to encapsulate a set of widgets in a logical structure.

There were no problems when writing this widget as it worked entirely as designed (that is, the logical structure was an extremely accurate representation).
