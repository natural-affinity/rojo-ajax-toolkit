Dialog Widget Documentation


The Dialog widget was designed in JavaScript and attempts to provide a simple, reusable mechanism for adding any number of dialog boxes to an HTML Web-page.

Internally, the JavaScript class stores the structure of the Dialog as an HTML DOM object. After this is appended to a Web-page you can see that this is a set of HTML tags combined in a special way to look like a Dialog. The motivation behind the widget was the lack modal dialog support within Mozilla Firefox. While this Dialog widget is not a true modal dialog, it attempts to emulate the same functionality by blocking access to the Web-page in the background. In terms of logical structure, the Dialog widget uses a ContentPane (which is composed of a TitleBar and Content Bar) to simulate a dialog box and an external HTML page for the backdrop.

Although JavaScript must be written to build the widget, it will ultimately be reduced to a set of HTML tags when you view the source code of the Web-page. Even though this widget contains another widget, the widgets still breakdown into their respective HTML structures. The Dialog guarantees that the HTML will be formatted correctly.

Include Mechanism

To use this JavaScript class in a Web-page, please add the HTML script tag with the appropriate attributes to the HEAD section of the HTML document. The path listed assumes that the file is located in the same directory as the HTML Web-page on which the widget will be used (although the include can be made in the BODY of the HTML document, it is recommended to place it in the HEAD section to reduce load times). 

<html>
  <head>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
    <script type=“text/javascript” src=“./class.dialog.js”></script>
  </head>
<html>

Browser Compatibility

This widget has been tested, and is supported on the following browsers:

  • Firefox 1.5.x
  • Firefox 2.0.x
  • Internet Explorer 6
  • Internet Explorer 7

Attributes

m_id

  The "id" given to the Dialog when rendered as HTML [Container Node]

m_container_class

  The name of the CSS class that will be given to the Dialog container [Container Node]

m_underlay_class

  The name of the CSS class that will be given to the Dialog underlay [Underlay Node]

m_content_pane

  The ContentPane object that will appear within the Dialog window

m_vscroll_width

  Stores the width of the browser vertical scrollbar. This is currently not used

m_hscroll_width

  Stores the width of the browser horizontal scrollbar. This is currently not used

m_ie_fix

  An internal counter used to handle the IE double resize() event, and invalid scroll event requests

m_hide_on_close

  A Boolean flag that hides the dialog when closed when true, and destroys it when false

m_closer

   Stores the Dialog close button as an HTML DOM structure

m_dom

   Stores the Dialog as an HTML DOM structure

Example Usage

The examples to follow are designed to serve as a simple walkthrough on how you would add the Dialog widget to a Web-page. Each example is designed to build on those before it, and therefore, the final example will show the overall picture regarding instantiating, customizing, building, displaying, and destroying the Dialog widget correctly. While it is not necessary to follow each step exactly, this will provide a general idea on how it works.

Example 1 - Hooking it up

First create an HTML hook or placeholder in which the widget will be placed on the Web-page. While each widget is self-contained, this is an easy way to map out the structure of the page without embedding lots of JavaScript within the HTML. "div" tags serve as great hooks, since they can be placed within almost any type of tag, and similarly, can contain almost any tag. Although this widget contains sub-widgets only one hook needs to be created for the parent component. 

<html>
  <head>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
    <script type=“text/javascript” src=“./class.dialog.js”></script>
  </head>
  <body>
    <div id=“dialog_hook”></div>
  </body>
<html>

Example 2 - Creating the Dialog

To create the Dialog, as in other object-oriented programming languages, the constructor must be called along with the use of the keyword new. Remember to create the sub-widgets required for the building the parent widget: the ContentPane (and its sub-widgets: TitleBar and ContentBar). 

<html>
  <head>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
    <script type=“text/javascript” src=“./class.dialog.js”></script>
    <script type=“text/javascript”>

        var widget = null;
        var pane_widget = null;
        var title_widget = null;
        var content_widget = null;

        function init()
        {
            content_widget = new ContentBar('my_content');
            title_widget = new TitleBar('my_title');
            pane_widget = new ContentPane('my_pane');
            widget = new Dialog('my_dialog');
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“dialog_hook”></div>
  </body>
<html>

Example 3 - Controlling Look and Feel

Some JavaScript must be used in order to customize the colors, borders, and size of the Dialog widget. See the class diagram and documentation to determine which methods are available for use. Rather than creating functions to set individual properties as with the ProgressBar widget, because there are more properties that can be set, CSS classes are used instead. These stylesheets can be external or internal. For demonstration purposes, internal stylesheet classes will be used. Remember to attach the sub-widgets to their parent widgets. Use the setHideOnClose() method to determine the dialog closing functionality. 

<html>
  <head>
    <style type=“text/css”>
      .label4
      {
          background-color:#3C5F84;
          color: #fff;
          font-family: verdana,arial,sans-serif;
          font-size:11px;
          font-weight:bold;
          padding-left:2px;
          height:22px;	
      }   
    </style>  
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
    <script type=“text/javascript” src=“./class.dialog.js”></script>
    <script type=“text/javascript”>

        var widget = null;
        var pane_widget = null;
        var title_widget = null;
        var content_widget = null;

        function init()
        {
            content_widget = new ContentBar('my_content');
            content_widget.setContent('Sample Dialog Content');

            title_widget = new TitleBar('my_title');
            title_widget.setContainerClass('label4');
            title_widget.setRowClass('label4');
            title_widget.setContent(['DIALOG TITLE']);

            pane_widget = new ContentPane('my_pane');
            pane_widget.setTitleBar(title_widget);
            pane_widget.setContentBar(content_widget);

            widget = new Dialog('my_dialog');
            widget.setContentPane(pane_widget);
            widget.setHideOnClose(false);
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“dialog_hook”></div>
  </body>
<html>

Example 4 - Displaying the Dialog

To use the widget in a Web-page the construct() method must be called in order to create the HTML DOM representation of the widget. This DOM object must then be appended to the hook that was previously created. The Dialog widget has a separate method, showDialog(), that must be called in order to make it visible. If the widget is going to be updated regularly, whether via AJAX or by having the user perform an action such as clicking a button, the construct() method must be called each time and the widget must be appended to the hook each time. showDialog() should also be called. Clear the hook of any previously appended Dialog widgets if you only want one to appear on the Web-page (that is, set the innerHTML property of the hook to an empty string). All sub-widget construct() method calls will be performed automatically. 

<html>
  <head>
    <style type=“text/css”>
      .label4
      {
          background-color:#3C5F84;
          color: #fff;
          font-family: verdana,arial,sans-serif;
          font-size:11px;
          font-weight:bold;
          padding-left:2px;
          height:22px;	
      }   
    </style>    
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
    <script type=“text/javascript” src=“./class.dialog.js”></script>
    <script type=“text/javascript”>

        var widget = null;
        var pane_widget = null;
        var title_widget = null;
        var content_widget = null;

        function init()
        {
            content_widget = new ContentBar('my_content');
            content_widget.setContent('Sample Dialog Content');

            title_widget = new TitleBar('my_title');
            title_widget.setContainerClass('label4');
            title_widget.setRowClass('label4');
            title_widget.setContent(['DIALOG TITLE']);

            pane_widget = new ContentPane('my_pane');
            pane_widget.setTitleBar(title_widget);
            pane_widget.setContentBar(content_widget);

            widget = new Dialog('my_dialog');
            widget.setContentPane(pane_widget);
            widget.setHideOnClose(false);

            var hook = document.getElementById('dialog_hook');
            hook.innerHTML = '';
            hook.appendChild(widget.construct());  
            widget.showDialog();            
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“dialog_hook”></div>
  </body>
<html>

Example 5 - Cleaning up Memory

To ensure that all memory associated with the widget is freed upon closing the Web-page or browser, it is necessary to call the destruct() method. While most programming languages have a destructor that is implicitly called, here, a explicit function call must be made. The hook to which the widget was appended should also be cleared. The destruct() method of the parent widget automatically calls the destruct() method of the sub-widgets. 

<html>
  <head>
    <style type=“text/css”>
      .label4
      {
          background-color:#3C5F84;
          color: #fff;
          font-family: verdana,arial,sans-serif;
          font-size:11px;
          font-weight:bold;
          padding-left:2px;
          height:22px;	
      }   
    </style>    
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
    <script type=“text/javascript” src=“./class.dialog.js”></script>
    <script type=“text/javascript”>

        var widget = null;
        var pane_widget = null;
        var title_widget = null;
        var content_widget = null;

        function init()
        {
            content_widget = new ContentBar('my_content');
            content_widget.setContent('Sample Dialog Content');

            title_widget = new TitleBar('my_title');
            title_widget.setContainerClass('label4');
            title_widget.setRowClass('label4');
            title_widget.setContent(['DIALOG TITLE']);

            pane_widget = new ContentPane('my_pane');
            pane_widget.setTitleBar(title_widget);
            pane_widget.setContentBar(content_widget);

            widget = new Dialog('my_dialog');
            widget.setContentPane(pane_widget);
            widget.setHideOnClose(false);

            var hook = document.getElementById('dialog_hook');
            hook.innerHTML = '';
            hook.appendChild(widget.construct());  
            widget.showDialog();            
        }

        function destroy()
        {
            widget.destruct();
            document.getElementById('dialog_hook').innerHTML = '';    
        }            
    </script>
  </head>
  <body onload=“init()” unload=“destroy()”>
    <div id=“dialog_hook”></div>
  </body>
<html>

Special Notes

  • This particular widget does not use default style in the absence of CSS classes.
  • Always call the destruct() method when unloading the page to prevent memory leaks.
  • Sub-widgets must be created and attached in order to use the widget correctly.
  • Sub-widget construct() and destruct() method calls are automatically managed by the parent (Dialog) widget.
  • Hiding the Dialog widget, whether explictly or by closing the dialog box, does not free the memory associated with it.

Lessons Learned

The Dialog widget was diffcult to create in that there were many cross-browser differences. Almost all of the properties, whether it be the width of the page or the distance that the scrollbar had been moved, are all stored in different variables. Also, the width of the scroll bars is different on each browser implementation. While it is possible to dynamically calculate these each time, the lag is quite noticeable. If the estimated scroll bar width is too small this can cause the page to scroll indefinitely.

Internet Explorer featured a bug whereby a "resize" window event would fire twice if the window size was changed. A large portion of the code in the fixDialog() method is dedicated to fixing this issue.

Most dialog implementations rely on the use of the zIndex (that is, foreground and background layering property) in order to make the rest of the page appear in the background. However, HTML drop-down boxes, have an issue in Internet Explorer where they are always in the foreground. Resolving this issue required the use of an external HTML page within an iframe tag as an underlay, rather than in a div tag.
