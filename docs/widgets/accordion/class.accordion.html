Accordion Widget Documentation


The Accordion widget was designed in JavaScript and attempts to provide a simple, reusable mechanism for adding any number of Accordion widgets to an HTML Web-page.

Internally, the JavaScript class stores the structure of the Accordion widget as an HTML DOM object. When the widget is appended to a Web-page, you can see that it is a set of HTML tags combined in a special way to look like an accordion. An Accordion widget is made up of any number of ContentPane widgets, each of which is composed of two-subcomponents: a TitleBar widget and ContentBar widget. When combined, these widgets appear as a series of headers with only a single section of content visible at any given time. Clicking on an inactive header (that is, whose accompanying content is not visible), produces the accordion effect (where the new content become visible and the old content becomes hidden).

Although JavaScript must be written to build the widget, it will ultimately be reduced to a set of HTML tags when you view the source code of the Web-page. Even though this widget contains other widgets, all widgets still breakdown into their respective HTML structures. The Accordion widget guarantees that the HTML will be formatted as a set of sections whose titles, when clicked, will minimize or maximize a new section of content.

Include Mechanism

To use this JavaScript class in a webpage, please add the HTML script tag with the appropriate attributes to the HEAD section of the HTML document. The path listed assumes that the file is located in the same directory as the HTML Web-page on which the widget will be used (although the include can be made in the BODY of the HTML document, it is recommended to place it in the HEAD section to reduce load times). The dependency widget classes (that is, the ContentPane, ContentBar, and TitleBar) must also be included. 

<html>
  <head>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
    <script type=“text/javascript” src=“./class.accordion.js”></script>
  </head>
<html>

Browser Compatibility

This widget has been tested, and is supported on the following browsers:

  • Firefox 1.5.x
  • Firefox 2.0.x
  • Internet Explorer 6
  • Internet Explorer 7

<div>                               <!-- Container Node -->

Attributes

m_id

  The "id" given to the Accordion when rendered as HTML [Container Node]

m_acc_class

  The name of the CSS that will be given to the Accordion [Container Node]

m_active_pane

  The index of the ContentPane that is currently active (that is, has visible content)

m_min_icon_src

  The URL to the icon used to show that a content pane is minimized (that is, the pane with its content not shown).
  This attribute is currently not used.

m_max_icon_src

  The URL to the icon used to show that a content pane is maximized (that is, the pane with its content shown).
  This attribute is currently not used.

m_icon_class

  The names of the CSS classes for the columns that hold the minimize or maximize icons appended to the TitleBars

m_panes

  An array of ContentPanes which comprise the Accordion

m_dom

  The Accordion as an HTML DOM structure

Example Usage

The examples to follow serve as a simple walkthrough on how you would add the Accordion widget to a Web-page. Each example is designed to build on those before it, and therefore, the final example will show the overall picture regarding instantiating, customizing, building, displaying, and destroying the Accordion widget correctly. While it is not necessary to follow each step exactly, this will provide a general idea on how it works.

Example 1 - Hooking it up

First create an HTML hook or placeholder in which the widget will be placed on the Web-page. While each widget is self-contained, this is an easy way to map out the structure of the page without embedding lots of JavaScript within the HTML. "div" tags serve as great hooks, since they can be placed within almost any type of tag, and similarly, can contain almost any tag. Although this widget contains sub-widgets, only one hook needs to be created for the parent component. 

<html>
  <head>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
    <script type=“text/javascript” src=“./class.accordion.js”></script>
  </head>
  <body>
    <div id=“accordion_hook”></div>
  </body>
<html>

Example 2 - Creating the Accordion

To create the Accordion, as in other object-oriented programming languages, the constructor must be called along with the use of the keyword new. Remember to create the sub-widgets required for the building the parent widget. For this example, ContentPane widgets are required along with their sub-widgets: TitleBars and ContentBars. 

<html>
  <head>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
    <script type=“text/javascript” src=“./class.accordion.js”></script>
    <script type=“text/javascript”>

        var widget = null;
        var pane_widget1 = null;
        var pane_widget2 = null;
        var title_widget1 = null;
        var title_widget2 = null;
        var content_widget1 = null;
        var content_widget2 = null;

        function init()
        {
            content_widget1 = new ContentBar('my_cont1');
            content_widget2 = new ContentBar('my_cont2');
            title_widget1 = new TitleBar('my_title1');
            title_widget2 = new TitleBar('my_title2');
            pane_widget1 = new ContentPane('my_pane1');
            pane_widget2 = new ContentPane('my_pane2');
            widget = new Accordion('my_accordion');
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“accordion_hook”></div>
  </body>
<html>

Example 3 - Controlling Look and Feel

Some JavaScript must be used in order to customize the colors, borders, and size of the Accordion, TitleBar, ContentBar, and ContentPane. See the class diagram and documentation to determine which methods are available for use. Rather than creating functions to set individual properties as with the ProgressBar widget, because the Accordion, TitleBar, ContentBar, and ContentPane have more properties that can be set, CSS classes are used instead. These stylesheets can be external or internal. For demonstration purposes, internal stylesheet classes will be used. The sample below will create an Accordion with two ContentPanes. 

<html>
  <head>
    <style type=“text/css”>
      .lfixed
      {
          width:758px;
      }

      .lfixed2
      {
          width:762px;
      }

      .label4
      {
          background-color:#3C5F84;
          color: #fff;
          font-family: verdana,arial,sans-serif;
          font-size:11px;
          font-weight:bold;
          padding-left:2px;
          height:22px;	
      }

      .accBody 
      {
          padding-top:10px;
          background: #C7E4C0;
          color: black;
          overflow: auto;
          height:140px;
      }

      .acc2
      {
          float: left;  
          width: 800px; 
          height: 250px;
          overflow: hidden;
      }      
    </style>
    <script type=“text/javascript” src=“./class.titlebar.js”></script>
    <script type=“text/javascript” src=“./class.contentbar.js”></script>
    <script type=“text/javascript” src=“./class.contentpane.js”></script>
    <script type=“text/javascript” src=“./class.accordion.js”></script>
    <script type=“text/javascript”>

        var widget = null;
        var pane_widget1 = null;
        var pane_widget2 = null;
        var title_widget1 = null;
        var title_widget2 = null;
        var content_widget1 = null;
        var content_widget2 = null;

        function init()
        {
            content_widget1 = new ContentBar('my_cont1');
            content_widget1.setContentClass('accBody');
            content_widget1.setContent('content section 1');

            content_widget2 = new ContentBar('my_cont2');
            content_widget2.setContentClass('accBody');
            content_widget2.setContent('content section 2');

            title_widget1 = new TitleBar('my_title1');
            title_widget1.setContainerClass('label4');
            title_widget1.setRowClass('label4');
            title_widget1.setColClass(['','lfixed']);
            title_widget1.setContent(['<img src=“./icon.bmp” alt=“” />',' Title1']);

            title_widget2 = new TitleBar('my_title2');
            title_widget2.setContainerClass('label4');
            title_widget2.setRowClass('label4');
            title_widget2.setColClass(['','lfixed2']);            
            title_widget2.setContent(['<img src=“./icon.bmp” alt=“” />',' Title2']);            

            pane_widget1 = new ContentPane('my_pane1');
            pane_widget1.setTitleBar(title_widget1);
            pane_widget1.setContentBar(content_widget1);

            pane_widget2 = new ContentPane('my_pane2');
            pane_widget2.setTitleBar(title_widget2);
            pane_widget2.setContentBar(content_widget2);

            widget = new Accordion('my_accordion');
            widget.setAccordionClass('acc2');
            widget.setContentPane([pane_widget1, pane_widget2]);
        }
    </script>
  </head>
  <body onload=“init()”>
    <div id=“accordion_hook”></div>
  </body>
<html>

Example 4 - Displaying the Accordion

To use the widget in a Web-page, the construct() method must be called in order to create the HTML DOM representation of the widget. This DOM object must then be appended to the hook that was previously created. This will allow the Accordion to become visible on the page. If the widget is going to be updated regularly, whether via AJAX or by having the user perform an action such as clicking a button, the construct() method must be called each time and the widget must be appended to the hook each time. It is important to clear the hook of any previously appended Accordion widgets if you only want one to appear on the Web-page (for example, set the innerHTML property of the hook to an empty string). All sub-widget construct() method calls will be performed automatically.

The destruct() method of the parent widget automatically calls the destruct() method of the sub-widgets.

Special Notes

  • This particular widget does not use default style information in the absence of CSS classes.
  • Always call the destruct() method when unloading the page to prevent memory leaks.
  • Sub-widgets must be created and attached in order to use the widget correctly.
  • Sub-widget construct() and destruct() method calls are automatically managed by the parent (Accordion) widget.
  • All icon related methods featured on the Accordion are still experimental and are not recommended for use.

Lessons Learned

No major problems were encountered while creating this widget with the exception of detecting mouse clicks from the user on each TitleBar, in order to know when to swap ContentPanes. This is because the "id" given to the TitleBar is used in order to determine if the user has requested to view another section of content.

The TitleBar is composed of numerous HTML tags and various forms of content (for example, text and images). Therefore, if the user happened to click the content instead of the HTML tag enacapsulating the content, then the swap functionality would not trigger (that is, the TitleBar had dead spots). To resolve this, the applyToAll() method was developed whereby the "id" given to the TitleBar container is applied to all nested HTML tags. This solution does not impact the use of document.getElementById() in order to find a particular widget on the Web-page because the use of this method will look for the widget with the specified "id", starting from the top of the page.
